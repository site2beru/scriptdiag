# scriptdiag

13.11.2025 uspensky.

Поиск источников проблемных скриптов/тегов/стилей на сайте

Если валидатор ругается на проблемные части кода, например, стили внутри div, незакрытые теги, отсутствующие или лишние атрибуты в тегах, сами лишние теги и т.п. — 
ищем их в файлах сайта, не перерывая все возможные шаблоны и компоненты вручную.

Протестировано на Netcat CMS, где шаблоны макетов сайта и компонентов разложены по разным папкам и файлам.
